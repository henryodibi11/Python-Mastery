# Design Patterns in Data Engineering

## Overview

Design patterns that solve real framework problems, focusing on patterns Odibi actually uses.

## Patterns Covered

1. **Registry Pattern** - Transform registry, function registry
2. **Factory Pattern** - `create_context`, connection factory
3. **Strategy Pattern** - Different engines, same interface
4. **Builder Pattern** - Config building
5. **Singleton Pattern** - When/when not to use
6. **Dependency Injection** - Managing dependencies
